Hi everyone,<br><br>Today, the ImageJDev team is proud to release the first beta of ImageJ2, version 2.0.0-beta1!<br>







<br>ImageJ <span><span>2.0</span></span>.<span><span>0</span></span>-beta1 is a &quot;beta&quot;-quality <span>release</span>, meaning the code is not finished. The design is more stable than before but still subject to change until the final 2.0.0 release. It is recommended that people continue to use ImageJ v1.x for critical work.<br>


<br>The ImageJ2 user interface is modeled after ImageJ v1.x as much as possible. However, under the hood, ImageJ2 is a complete redesign of ImageJ. It provides backward compatibility with older versions of ImageJ by bundling the latest v1.x code and translating between &quot;legacy&quot; and &quot;modern&quot; image structures.<br>


<br>The beta still has many bugs and limitations, but we have achieved several substantial milestones:<br><ul><li>Support for most ImageJ1 plugins and macros</li><li>Many new pixel types (thanks to ImgLib2)</li><li>Multidimensional data beyond 5D (thanks to ImgLib2)</li>


<li>Import and export of all SCIFIO-supported file formats</li><li>Improved region of interest (ROI) tools</li><li>Truly headless: image processing is separate from user interface</li><li>Automatic updates: once you download beta1, ImageJ can update itself</li>


<li>Easily install additional plugins, including Fiji (over 350 plugins!)</li><li>175 new and/or reimplemented core ImageJ2 plugins so far</li><li>125+ bugs fixed</li></ul><br>Many legacy ImageJ plugins and macros work; many others do not. There are still many bugs. In particular, there are a few issues specifically regarding 2.0.0-beta1:<br>


<ul><li>On Windows &amp; Linux, you must have Java already installed; ImageJ2 is not bundled with Java, nor will it download it for you.</li><li>On Mac OS X 10.5 Leopard, there is a bug that prevents ImageJ2 from launching successfully. This bug will be fixed in the next beta release, and is already fixed in the development version.</li>


<li>On Windows, when ImageJ2 first starts, a console message (&quot;Falling back to system Java&quot;) appears, which is merely informative and is safe to ignore. This message will no longer be displayed in the next beta release, and is already fixed in the development version.</li>


</ul><br>For further details on this release, including a discussion of new features, please see our blog post:<br>  <a href="http://developer.imagej.net/2012/04/09/imagej-v200-beta1" target="_blank">http://developer.imagej.net/2012/04/09/imagej-v200-beta1</a><br>
