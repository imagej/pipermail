Hi everyone,<br><br>As you know, we are currently documenting the ImageJ2 development work at:<br>  <a href="http://imagejdev.org/">http://imagejdev.org/</a><br><br>However, we do have the domain <a href="http://imagej.net">imagej.net</a>. Currently, it is in use as follows:<br>

  <a href="http://imagej.net">imagej.net</a> -&gt; <a href="http://imagej.nih.gov/ij">imagej.nih.gov/ij</a><br>  <a href="http://developer.imagej.net">developer.imagej.net</a> -&gt; <a href="http://imagejdev.org">imagejdev.org</a><br>

  <a href="http://fiji.imagej.net">fiji.imagej.net</a> -&gt; <a href="http://fiji.sc">fiji.sc</a><br>  <a href="http://test.imagej.net">test.imagej.net</a> -&gt; a test website, not populated with anything<br><br>I think that moving forward, we should phase out the <a href="http://imagejdev.org">imagejdev.org</a> domain, which has been more targeted toward our initial 2-year effort than to the ImageJ project as a whole. The content from <a href="http://imagejdev.org">imagejdev.org</a> would remain in place, but served under <a href="http://developer.imagej.net">developer.imagej.net</a> instead (with an automatic redirect from the old <a href="http://imagejdev.org">imagejdev.org</a> of course). This is in line with other organizations having a &quot;developer&quot; subdomain or similar: <a href="http://developer.mozilla.org">developer.mozilla.org</a>, <a href="http://developer.amazon.com">developer.amazon.com</a>, <a href="http://developers.google.com">developers.google.com</a>, etc.<br>

<br>For source code repositories, Trac, Jenkins, Maven and other resources, we could have a new subdomain such as <a href="http://code.imagej.net">code.imagej.net</a> that would serve the purpose of the current <a href="http://dev.imagejdev.org">dev.imagejdev.org</a>. (We could call it &quot;<a href="http://dev.imagej.net">dev.imagej.net</a>&quot; but that might be confusing since there would also be a &quot;<a href="http://developer.imagej.net">developer.imagej.net</a>&quot; that is something else.)<br>

<br>So, in summary, my proposal for updating the web resources is:<br>  <a href="http://imagej.net">imagej.net</a> = Top level ImageJ portal. About ImageJ, downloads, plugins, etc.<br>  <a href="http://developer.imagej.net">developer.imagej.net</a> = Documentation for developers coding against ImageJ.<br>

  <a href="http://code.image.net">code.image.net</a> = Project management tools: VCS, Trac, Jenkins, Maven<br><br>Thoughts? Suggestions?<br><br>-Curtis<br><br>
