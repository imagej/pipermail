I know this thread is several weeks old, but in the interest of avoiding
hanging threads, I'm replying now. Lee & I & others spoke at length about
the directions for headless image support, and Lee has made lots of
progress along these lines in a branch. I then hacked up the branch some
more, and Lee did some more work... suffice to say, the branch is here:

    https://github.com/imagej/imagej/tree/lee_headless-ctr

We are in the process of getting it merged back to trunk/master, at which
point the support for headless displays will be much better. This work
should be complete in time for the beta3 release before the end of May.
