Changes:

[Mark Hiner] Added support for ImgSaver#isCompressible

The SaveAsImage plugin now takes advantage of ImgSaver#isCompressible to
check its ImgPlus for potential compression requirements (of unknown
Axes of size > 2).

If compression will be required, which necessitates a change in the
ImgPlus structure, the user will be warned and provided a YES/NO dialog
option.

[Mark Hiner] SaveAs now Warns when overwriting

Previously a QUESTION type message was displayed to the user when
potentially overwriting an existing file. It now is a WARN type message.

[Mark Hiner] Added label to SaveAs dialog

The SaveAs... dialog box now says "File to save" instead of "OutputFile"
with no space.

[Mark Hiner] Save/Open Image display proper Error messages

When the Open and SaveAs plug-ins are handling errors, they now display
ERROR_MESSAGE type messages instead of the default type.

[Mark Hiner] Added initializer to SaveAsImage

To avoid persisting the Save paths in SaveAsImage plugin, we set the
File parameter to persist = false and instead use an initializer to
attempt to pull in the ImgPlus's source, if present.

[Mark Hiner] On errors, display dialog after logging

In case there is an exception during dialog display, we log first and
then display the errors received in the SaveAsImage plugin.

[Mark Hiner] Display errors on UI

OpenImage and SaveAsImage plug-ins now display caught error messages via
uiService.showDialog() when handling exceptions.

[Mark Hiner] Added AbstractImageHandler

AbstractImageHandler is a parent for OpenImage and SaveAsImage plug-ins.
It contains the StatusListener#statusUpdated() implementation that can
be shared between these and other plug-ins.

[Mark Hiner] Updated OpenImage plugin to use ome.scifio.img.*

Updated references from net.imglib2.io to ome.scifio.img for ImgOpener
and ImgIOException.

[Mark Hiner] Added StatusUpdates to SaveAsImage

The SaveAsImage plugin will now report status messages, following the
same conventions as the OpenImage plugin.

Note that SaveImage does not report statuses as it always delegates to
SaveAsImage.
