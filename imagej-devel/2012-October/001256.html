Note that "IJ" in ImageJ2 scripting refers to the current ImageJ context,
i.e. an instance of imagej.ImageJ. It does *not* refer to the static class
ij.IJ -- which would not make too much sense anyway, because we are most
likely not able to support too many of that class' functions.
