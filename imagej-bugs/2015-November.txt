but at least its a simple fix.

--- Comment #1 from Wayne Rasband <wsr at nih.gov> ---
The ij.process.AutoThresholder class currently only works with 8-bit
histograms. To threshold a 16-bit image, ImageJ converts the image to 8-bits,
gets the threshold and then scales it to a 16-bit threshold. This JavaScript
demonstrates how to get the thresholds of each image in a 16-bit stack.

    ip.setAutoThreshold("Moments dark");
    print(i+" "+ip.getMinThreshold());
  }

When you use 'Edit > Draw' instead, you get a "Process stack?" dialog, and

the case for the 'Fill' command, where both 'F' and 'Edit > Fill' will record:

    run("Fill", "slice");

You can work around this problem by upgrading to the latest ImageJ daily build
(1.50e36) and enabling the "Use JFileChooser" option. Here is an example:

  setOption("JFileChooser", true);
  dir = getDirectory("Choose a Directory");
  setOption("JFileChooser", false);

Unlike the native directory chooser, JFileChooser still has a title bar on OS X
10.11 (El Capitan).
