        <pre>After updating Fiji today, it became terribly slow when using the Channes tool.
Up to the point where it would freeze entirely. 
Having done a lot of back and forth and updating my (Oracle) Java to 1.8.0_51,
I could still not fix the problem. Therefore, I downgraded to the current
Life-Line version from Nov 25, 2014 and everything was working again. 
I then upgraded all components step by step and found that updating
"Contents/MacOS/ImageJ-macosx" (released Jul 15, 2015) causes all the trouble.
Without this update, Fiji is running Java 1.6.0_65 according to the status bar
(despite 1.8.0_51 still being installed and also being shown in Terminal when
using the "java -version" command). After updating
"Contents/MacOS/ImageJ-macosx", the status bar shows Java 1.8.0_51, but Fiji
exhibits the deficits mentioned above.

If you need any further information, please let me know.

Glad for any help on this issue (although for now I found a way to work again).
