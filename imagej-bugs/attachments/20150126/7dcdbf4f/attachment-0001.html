        <pre>"Use SCIFIO when opening files" was turned off.
Turning it on opens the 16bit sample images as 8-bit - but shows them as empty
(white) images.
This is true for "M51 Galaxy" and "CT" (both 16bit), but not "Gel" (8bit) which
opens fine with SCIFIO.
"T1 Head" (16bit) doesn't open at all with SCIFIO.

Furthermore: 
Drag & drop of /Applications/Fiji.app/samples/m51.zip 
opens as correct 16bit image with SCIFIO turned off, 
and opens as all-white 8bit image with SCIFIO turned on.

So, obviously, Fiji can read and open a 16bit image correctly - only the File >
Open Samples function seems to have some issues.
Not crucial, just disturbing, as this is (now, used to be)  part of our
teaching examples for image processing using Fiji here at ETH.
