        <pre>You have several update sites enabled. The "NoSuchMethodError" indicates that
one (or more) of them is shipping an old version of the core ImgLib2 library.
This situation is known as "version skew". See:

  <a href="http://imagej.net/Troubleshooting#NoSuchMethodError_or_NoClassDefFoundError">http://imagej.net/Troubleshooting#NoSuchMethodError_or_NoClassDefFoundError</a>

The problem in this case is the Alex-krull update site. Try disabling it.

If you need the functionality available from that update site, you can keep a
separate copy of ImageJ with just that one update site enabled, and launch it
separately as needed.

You may also want to contact him via email about the problem, and ask him to
stop shadowing these old versions of core ImageJ libraries:

  <a href="http://imagej.net/User:Alex-krull">http://imagej.net/User:Alex-krull</a></pre>
