        <pre>Can you check your temporary directory[1] and see if there is a file called
"ImageJ-xxxx-_0-7.stub", where "xxxx" is your user name?

If so, try deleting that file and restarting Fiji.

This file is created by the "Single Instance Listener" function of ImageJ, to
ensure only one instance is running at a time. I think there is a bug where it
isn't deleted after updating, so when you restart afterwards it tries to
connect to a "running instance" that doesn't actually exist.

If you're not able to find this file or directory please let me know.
