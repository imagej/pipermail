        <pre><span class="quote">> In WCIF ImageJ there were a range of plugins for looking at
> colocalisation but these appear not to have been implemented in Fiji.</span >

MBF ImageJ shipped with two colocalization plugins:

* <a href="http://fiji.sc/Colocalization_Threshold">http://fiji.sc/Colocalization_Threshold</a>
* <a href="http://fiji.sc/Colocalization_Test">http://fiji.sc/Colocalization_Test</a>

Both are still shipped with Fiji, but they are deprecated. We recommend using
the newer Coloc 2 plugin instead:

* <a href="http://fiji.sc/Coloc_2">http://fiji.sc/Coloc_2</a>

Also, if you haven't read it yet, please read the Colocalization Analysis guide
from the Fiji Cookbook:

* <a href="http://fiji.sc/Colocalization_Analysis">http://fiji.sc/Colocalization_Analysis</a>

<span class="quote">> Coloc2 seems unstable and is also not able to analyse colocalisation
> data the way we need</span >

You will have to be more specific. I suggest sending a mail to the ImageJ
mailing list (<a href="http://imagej.net/Mailing_Lists">http://imagej.net/Mailing_Lists</a>), so that one of the Coloc 2
plugin maintainers can assist you.

<span class="quote">> Further since we have a lot of data we would like to script our
> analysis but the colocalisation plugins seem not to get on well with
> the macro writing. </span >

Coloc 2 is supposed to work from a macro. If not, again, please write to the
ImageJ list with a specific description of your issue.

Here is a guide to writing good ImageJ issue reports:

* <a href="http://imagej.net/Bug_reporting_best_practices">http://imagej.net/Bug_reporting_best_practices</a>

I am closing this issue, since it does not raise any specific bugs, but please
feel welcome to contact the community mailing lists for assistance. There are
many friendly people on the ImageJ mailing list.</pre>
