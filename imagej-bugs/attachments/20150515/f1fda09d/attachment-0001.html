        <pre>I am trying to open all series from tiling/stitching expt in a Leica LIF file.
In bioformats, I select 'open all series' and 'select range' option, then pick
one channel, #3, in all series. [arduous task] The extracted names are not the
series name/number seen in preview GUI, but are all identical, the name of the
LIF, and thus impossible to see the series order via titles.
IF there are more than 20 or so series, the order of fiji windows no longer
matches the preview GUI order and there is no way to tell when the windows are
no longer in sync with collection order.
I need to process the tiles in collection order so they can be re-stitched. My
macros work for small LIF runs, but yield scrambles if there are many tiles.  
If the series import window order in fiji would remain in the order in the
setup GUI, all is well. 
If the tile import windows used the series name vs the LIF name for tiled data,
I would also be in good shape. --Every window has the name; LIFname -
TileScan_001 [54 of them in present case!]. The FIJI taskbar WINDOWS pulldown
is hopeless to select a particular series as well. [all the same name, no index
number]
The TileScan LIF files use a  _s## format in their own tif export output; this
is different vs regular LIF data. Perhaps this is a reason for the hiccups, or
perhaps this is a WIndows7 thing. BUT I suspect there is more to this than just
mixed file order since too many of the windows seem to lack data clearly
present within LASAF software that generated it. I see too many black windows
PLUS windows out of capture order.
Source files are big, but can upload LIF, if desired.
