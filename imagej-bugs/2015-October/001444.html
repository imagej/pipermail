Thanks for the update!

The fact that the batch file does not illustrate this problem strongly suggests
that this issue is specifically related to the ImageJ native launcher somehow.

The good news is: we are planning to retire that launcher soon in favor of
JavaFX-packaged application bundles. It is likely (but not certain) that those
will not have suffer from the same problem.

The bad news is: it will be a couple more months (at least) before that
transition can be completed. In the interim, you could use that batch file to
avoid the issue, although you probably want to edit it to (at minimum) add
"-Xmx2g" for however many gigs of memory you want ImageJ to use.

I am assigning this issue to myself, and will follow up as soon as we have a
new application bundle for you to test out.
