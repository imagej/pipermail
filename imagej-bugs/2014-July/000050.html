In contrast to ImageJ 1.x, ImageJ2 does not have the single-instance listener.
But you can open multiple images at the same time in which case the same
instance will open them all.

As to the suffix: this is intentional, as ImageJ 1.x' macro language frequently
identifies the current image *by name*.
