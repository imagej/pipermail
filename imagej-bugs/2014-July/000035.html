--- Comment #1 from Curtis Rueden <ctrueden at wisc.edu> ---
Thanks for the report. This bug is my fault, for including that command with a
shortcut of "Ctrl+\" when animation already has "\". For better and worse,
ImageJ lets you type the Ctrl+<X> shortcuts as just <X>. Very handy most of the
time, but causing a clash in cases like this.

I will think about how best to fix it. I would ideally like to keep Dump Stack
as Ctrl+\, and _only_ Ctrl+\, _not_ Cmd+\ on OS X nor \ without modifiers.
Perhaps we can improve the ImageJ2 plugin framework to enable this.
