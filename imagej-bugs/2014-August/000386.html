Hello!

Problematic statement:
IJ.run("Duplicate...","new_name")

Above statement should duplicate an entire image stack and call it "new_name".
Fiji only duplicates the first slice of the stack and ignores the "new_name"
parameter.

The corresponding macro statement:
run("Duplicate...","new_name");
does work as intended.

The Jython statement has same problems when run from the interactive Jython
Interpreter.

Any help in fixing this bug will be much appreciated.
