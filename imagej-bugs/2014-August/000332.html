Dear Johannes and everyone else, 
I’m trying to use the rigid registration plugin you developed to register
(translation + rotation) images of the same sample acquired with different
cameras on the same microscope. More specifically, I have a stack of images of
multi-color beads that I would like to use as reference to calculate the shift
matrix and then apply it to the real image stack of my sample. Your plugin
seems perfectly suited for the purpose, however, I’m having troubles because
the plug-in produces mistakes when outputting the transformed image stack.
While the first image of the transformed stack looks good the following ones
are a stitch of the subsequent images in the stack (e.g. slice 2 would be a
stitch of slice 1 and 2, slice 3 a stitch of slices 2 and 3 and so on). The
problem seems to be not systematic as for example I have it if I try to
transform stack-1 to match stack-2 but not viceversa. Moreover, the stitched
transformed stack looks different using different parameters (i.e. the stitched
regions change). This looks like a bug in the plugin If you need I can send you
an example of the wrong transformed output along with the source image stacks.
Please let me know how I can help you with this. 
Thanks a lot in advance for your help. Claudio
