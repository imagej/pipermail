When running 'Image > Properties...' on an image that already has a scaling in
µm set, the macro recorder will issue a warning:
<<warning: the options string contains one or more non-ascii characters>>

I suggest a more intelligent handling by either changing the prefilled input
field content or the recorded parameter to "um" when a previous µm scaling is
detected.
